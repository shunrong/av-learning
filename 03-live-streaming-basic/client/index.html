🎥 Node.js 直播系统

基于 RTMP + HTTP-FLV 的完整直播方案

🚀 快速开始

安装 FFmpeg（推流工具）

brew install ffmpeg

准备测试视频

找一个任意 .mp4 视频文件，放到项目根目录，命名为 test.mp4

推流到服务器

ffmpeg -re -i test.mp4 -c copy -f flv rtmp://localhost:1935/live/stream

💡 看到 "Stream mapping" 和不断滚动的日志就说明推流成功了！

打开播放器观看

▶️ 打开播放器

📖 核心概念

推流地址

rtmp://localhost:1935/live/stream ↓ ↓ ↓ ↓ 服务器 端口 应用名 流名称

live: 应用名（可自定义）
stream: 流名称（可自定义，播放时要对应）

播放地址

HTTP-FLV: http://localhost:8080/live/stream.flv

浏览器通过 flv.js 库解析 FLV 格式的直播流

延迟说明

HTTP-FLV 延迟通常在 3-5 秒

对比你之前学的 WebRTC（<500ms），这个延迟较高，但支持更多观众

🧪 实验建议

1. 多观众测试

同时打开 3-5 个浏览器标签页播放，观察服务器日志中的观众连接

2. 体验延迟

用摄像头推流，对着镜头挥手，观察播放器中的延迟

ffmpeg -f avfoundation -i "0:0" -c:v libx264 -preset ultrafast -f flv rtmp://localhost:1935/live/webcam

3. 对比 WebRTC

回顾你的 02-webrtc-mesh-conference 项目，对比两者的延迟和适用场景

❓ 常见问题

Q: 推流后播放器黑屏？
A: 等待 3-5 秒缓冲，检查推流地址和播放地址的流名称是否一致
Q: FFmpeg 命令不存在？
A: 需要先安装：brew install ffmpeg
Q: 如何停止推流？
A: 在 FFmpeg 运行的终端按 Ctrl+C
